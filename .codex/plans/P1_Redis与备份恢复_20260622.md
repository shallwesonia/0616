## 风险点

- Redis 不能成为权威数据源，缓存故障不得造成 PostgreSQL 数据丢失。
- Redis Key 必须包含 Workspace 边界。
- 消息缓冲必须有长度和 TTL 上限。
- WebSocket 缓存读取失败必须回退到 PostgreSQL，且健康检查应暴露故障。
- 备份恢复命令不得自动覆盖现有数据库。

## 待确认问题

- 无阻塞问题。采用 PostgreSQL 权威存储、Redis 可重建辅助缓存；Docker 默认启用 Redis，本地 JSON 模式默认禁用。

## 回滚方案

- 设置 `REDIS_ENABLED=false`，Platform API 继续使用 PostgreSQL。
- 恢复上一版 Platform API 镜像，不删除 Redis 或 PostgreSQL Volume。
- 备份恢复仅对新建隔离验证库执行，不覆盖当前运行库。
