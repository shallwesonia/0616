# 1～20 Workspace 方案文档统一优化

## 任务背景

现有平台已具备二维地图、机器狗 command/result MQTT、消息筛选、Docker 部署和基础联调能力。下一阶段需要统一引入多用户、每 Workspace 独立运行栈，以及 Task、Plan、Action、Observation、CurrentState、Snapshot、Trace 和仿真驾驶舱。

## 用户需求

- 按 1～20 个 Workspace 的规模优化全部方案文档。
- 每个用户默认拥有独立 Workspace 和独立前后端运行环境。
- 增加仿真驾驶舱专项规划。
- 本次只修改方案文档，不修改业务代码。

## 当前文档现状

- 已有实施、路线图、数据库、通信、连接和机器狗 MQTT 共 6 份正式文档。
- 当前部署描述以单 Workspace Docker Compose 为主。
- Task、Plan、Action、Observation、CurrentState、Snapshot、Trace 尚未形成统一模型。
- 尚无仿真驾驶舱专项实施文档。

## 风险点

- Workspace 隔离不得破坏既有机器狗 MQTT Topic 格式。
- 每 Workspace 独立前后端不能演变为复制源码和手工维护多套版本。
- Task、Plan、Action 等概念必须职责单一，避免状态、日志和快照重复存储。
- 1～20 Workspace 应优先 Docker Compose 和轻量控制平面，不提前引入 Kubernetes。
