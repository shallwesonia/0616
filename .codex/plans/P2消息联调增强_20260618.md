# P2 消息联调增强

## 任务背景

机器狗 MQTT command/result 协议已经切换完成，下一步需要提升真实设备联调时的消息定位、契约查看和问题诊断能力。

## 用户需求

- 继续下一步搭建。
- 沿用既有开发规划。
- 强化连接、协议、消息总成和真实设备接入准备。

## 当前代码现状

- 已支持 `factory/dogs/{robotCode}/command`。
- 已支持 `factory/dogs/{robotCode}/result`。
- 已支持 `/api/v1/connections` 和 `/api/v1/mqtt/contract`。
- 消息列表当前只能按 `limit` 查询，缺少联调筛选能力。
- 前端只展示最近消息，未展示完整契约和筛选入口。

## 风险点

- 消息筛选不能破坏现有 `/api/v1/messages?limit=` 兼容行为。
- 前端筛选必须不影响 WebSocket 实时刷新。
- 契约展示只做只读，不允许前端直接下发 MQTT。
