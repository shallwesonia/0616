## 1. 终版结论

平台和机器狗之间只保留两个 MQTT Topic：

```text
factory/dogs/{robotCode}/command
factory/dogs/{robotCode}/result
```

终版规则：

1. command topic 只承载 `messageType=command` 的 JSON 消息体。
2. result topic 只承载 `messageType=event` 的 JSON 消息体。
3. `{robotCode}` 必须与消息体中的 `robotCode` 一致。
4. 同一 Workspace 内 `robotCode` 必须唯一，多机器人同时在线时每台机器人使用独立 MQTT clientId。
5. 第一阶段固定支持 `goto_pose`、`stop`、`where` 三类命令。
6. 任务类命令使用 `taskId`，查询类命令使用 `requestId`。
7. 命令相关 result 必须沿用 command 中的 `commandId`、`taskId`、`traceId`。
8. 查询相关 result 必须沿用 command 中的 `commandId`、`requestId`、`traceId`。
9. 错误事件必须携带 `error` 对象。

## 2. command 通用结构

```json
{
  "schemaVersion": "1.0",
  "messageType": "command",
  "commandId": "CMD-20260528-000001",
  "taskId": "TASK-20260528-000001",
  "requestId": null,
  "robotCode": "DOG-001",
  "traceId": "TRACE-20260528-000001",
  "command": "goto_pose",
  "issuedAt": "2026-05-28T10:00:00+08:00",
  "timeoutMs": 120000,
  "operatorId": "scheduler",
  "source": "scheduler",
  "params": {
    "x": 12.34,
    "y": 56.78
  }
}
```

## 3. command 字段定义

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `schemaVersion` | string | 是 | 固定 `1.0` |
| `messageType` | string | 是 | 固定 `command` |
| `commandId` | string | 是 | 单次命令唯一 ID |
| `taskId` | string/null | 任务类必填 | `goto_pose`、任务内 `stop` 必填 |
| `requestId` | string/null | 查询类必填 | `where` 必填 |
| `robotCode` | string | 是 | 机器狗编码，必须与 Topic 一致 |
| `traceId` | string | 是 | 链路追踪 ID |
| `command` | string | 是 | 命令名 |
| `issuedAt` | string | 是 | 平台下发时间，ISO 8601 |
| `timeoutMs` | number | 建议 | 超时时间 |
| `operatorId` | string/null | 否 | 操作人或系统调用方 |
| `source` | string | 是 | `api`、`scheduler`、`agent`、`system` |
| `params` | object | 是 | 命令参数 |

## 4. 命令清单

| command | 用途 | `taskId` | `requestId` | 设备预期响应 |
|---|---|---|---|---|
| `goto_pose` | 移动到指定坐标或点位 | 必填 | null | `command.accepted/rejected`、`task.started`、`task.succeeded/failed` |
| `stop` | 停止当前任务 | 必填 | null | `command.accepted/rejected`、`task.stopped` |
| `where` | 查询当前位置 | null | 必填 | `where.result` 或 `where.failed` |

## 5. goto_pose

```json
{
  "schemaVersion": "1.0",
  "messageType": "command",
  "commandId": "CMD-20260528-000001",
  "taskId": "TASK-20260528-000001",
  "requestId": null,
  "robotCode": "DOG-001",
  "traceId": "TRACE-20260528-000001",
  "command": "goto_pose",
  "issuedAt": "2026-05-28T10:00:00+08:00",
  "timeoutMs": 120000,
  "operatorId": "scheduler",
  "source": "scheduler",
  "params": {
    "x": 12.34,
    "y": 56.78,
    "z": 0,
    "yaw": 1.57,
    "mapId": 1,
    "positionCode": "UNLOAD_POINT_B",
    "speedLimit": 1.2
  }
}
```

`params` 字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `x` | number | 是 | 目标 x 坐标 |
| `y` | number | 是 | 目标 y 坐标 |
| `z` | number | 否 | 目标 z 坐标，默认 0 |
| `yaw` | number | 否 | 目标朝向 |
| `mapId` | number/string | 否 | 地图 ID |
| `positionCode` | string | 否 | 业务点位编码 |
| `speedLimit` | number | 否 | 速度限制 |

## 6. stop

`stop` 必须生成新的 `commandId`，但应复用被停止任务的 `taskId` 和 `traceId`。

```json
{
  "schemaVersion": "1.0",
  "messageType": "command",
  "commandId": "CMD-20260528-000002",
  "taskId": "TASK-20260528-000001",
  "requestId": null,
  "robotCode": "DOG-001",
  "traceId": "TRACE-20260528-000001",
  "command": "stop",
  "issuedAt": "2026-05-28T10:01:00+08:00",
  "timeoutMs": 10000,
  "operatorId": "scheduler",
  "source": "scheduler",
  "params": {
    "reason": "manual_stop",
    "previousCommandId": "CMD-20260528-000001"
  }
}
```

## 8. result 通用结构

```json
{
  "schemaVersion": "1.0",
  "messageType": "event",
  "event": "task.succeeded",
  "eventId": "EVT-20260528-000001",
  "commandId": "CMD-20260528-000001",
  "taskId": "TASK-20260528-000001",
  "requestId": null,
  "robotCode": "DOG-001",
  "traceId": "TRACE-20260528-000001",
  "source": "device",
  "timestamp": "2026-05-28T10:01:20+08:00",
  "data": {
    "result": 1,
    "x": 12.34,
    "y": 56.78,
    "yaw": 1.57,
    "battery": 80
  },
  "error": null
}
```

## 9. result 字段定义

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `schemaVersion` | string | 是 | 固定 `1.0` |
| `messageType` | string | 是 | 固定 `event` |
| `event` | string | 是 | 标准事件名 |
| `eventId` | string | 建议 | 事件唯一 ID |
| `commandId` | string/null | 命令相关必填 | 关联平台下发命令 |
| `taskId` | string/null | 任务相关必填 | 关联业务任务 |
| `requestId` | string/null | 查询相关必填 | 关联查询请求 |
| `robotCode` | string | 是 | 机器狗编码 |
| `traceId` | string | 是 | 沿用 command 的 `traceId`；周期位置可用设备级 trace |
| `source` | string | 是 | 设备回传固定 `device` |
| `timestamp` | string | 是 | 事件发生时间，ISO 8601 |
| `data` | object | 是 | 事件数据 |
| `error` | object/null | 否 | 错误对象，无错误时为 null |

## 10. 事件清单

| event | 用途 | 关联字段 | 数据 |
|---|---|---|---|
| `command.accepted` | 设备接受命令 | `commandId` | `data.acceptedAt` |
| `command.rejected` | 设备拒绝命令 | `commandId` | 必填 `error` |
| `task.started` | 任务开始 | `commandId/taskId` | 可带当前 `x/y/yaw/battery` |
| `task.succeeded` | 任务成功 | `commandId/taskId` | `data.result=1`、最终位置、电量 |
| `task.failed` | 任务失败 | `commandId/taskId` | 必填 `error`，可带 `result=0` |
| `task.stopped` | 停止完成 | `commandId/taskId` | 可带 `reason` |
| `task.timeout` | 任务超时 | `commandId/taskId` | 必填 `error`，可带 `timeoutMs` |
| `pose.updated` | 周期位置和电量，也是心跳 | `robotCode` | `x/y/z/yaw/battery` |
| `where.result` | where 成功 | `commandId/requestId` | `x/y/z/yaw/battery` |
| `where.failed` | where 失败 | `commandId/requestId` | 必填 `error` |
| `device.offline` | 平台判定离线 | `robotCode` | 必填 `error`，可带 `lastSeenAt` |

## 11. error 对象

```json
{
  "errorCode": "NAVIGATION_NO_PATH",
  "errorMessage": "目标点不可达",
  "detail": "目标点附近不可通行",
  "retryable": false,
  "source": "device"
}
```

字段要求：`errorCode`、`errorMessage`、`retryable`、`source` 必填；`detail` 可选。

首批错误码：

```text
DEVICE_BUSY
INVALID_TARGET_POSE
NAVIGATION_NO_PATH
NAVIGATION_BLOCKED
LOW_BATTERY
COMMAND_ACK_TIMEOUT
TASK_TIMEOUT
STOP_FAILED
DEVICE_OFFLINE
MQTT_BROKER_UNAVAILABLE
WHERE_FAILED
```

## 12. 回传校验

| 检查项 | 要求 |
|---|---|
| topic | 必须是 `factory/dogs/{robotCode}/result` |
| `messageType` | 必须是 `event` |
| `robotCode` | Topic 与消息体一致 |
| `commandId` | 命令相关事件必须携带 |
| `taskId` | 任务相关事件必须携带 |
| `requestId` | where 相关事件必须携带 |
| `traceId` | 命令响应沿用 command 的 `traceId` |
| `error` | 错误事件必须携带，非错误事件为 null |

## 13. REST 兼容说明

平台 REST `POST /api/v1/commands` 支持两类输入：

- 新格式：`robotCode`、`command`、`params`。
- 兼容格式：`robotId`、`commandType`、`target`。

兼容映射：

| REST 输入 | MQTT 输出 |
|---|---|
| `robotId` | `robotCode` |
| `commandType=move` | `command=goto_pose` |
| `commandType=carry` | `command=goto_pose` |
| `target` | `params` |

设备侧只以 MQTT command/result 协议为准，不依赖 REST 兼容字段。

## 14. Workspace 隔离标准

1～20 Workspace 阶段采用每 Workspace 独立 MQTT Broker。

因此 Topic 保持不变：

```text
factory/dogs/{robotCode}/command
factory/dogs/{robotCode}/result
```

禁止在终版 Topic 或 Payload 中增加以下字段：

```text
tenantId
userId
workspaceId
runId
```

隔离信息由平台连接配置、Broker 实例和数据库上下文管理，不传递给机器狗固件。

规则：

- `robotCode` 在单个 Workspace 内唯一。
- 不同 Workspace 可以使用相同 `robotCode`，因为 Broker 完全独立。
- 真实机器狗同一时间只能绑定一个 Workspace。
- 每个机器狗使用独立用户名或客户端证书。
- ACL 只允许订阅自身 command、发布自身 result。
- Workspace 解绑或切换时必须撤销旧 MQTT 凭证。

当前开发环境默认编队：

| robotCode | MQTT clientId | 默认起点 |
|---|---|---|
| `robot-001` | `virtual-dog-robot-001` | `x=220,y=360` |
| `robot-002` | `virtual-dog-robot-002` | `x=320,y=360` |
| `robot-003` | `virtual-dog-robot-003` | `x=420,y=360` |

平台只向已登记的 `robotCode` 发布 command；未知 `robotCode` 的 Action 必须被拒绝。

## 15. 统一运行模型映射

| 平台对象 | MQTT 对象 | 映射规则 |
|---|---|---|
| Task | command.taskId | 任务类 command 必须携带 |
| Plan Step | 无直接字段 | 由平台 Action 记录关联，不扩展设备协议 |
| Action | command | 一个 Action 对应一次设备执行尝试 |
| Action.commandId | command.commandId | 每次下发唯一 |
| Action.requestId | command.requestId | where 查询使用 |
| Trace | command/result.traceId | command 和 result 必须沿用 |
| Observation | result event | 每条合法 result 生成一条 Observation |
| CurrentState | pose.updated 等 result 聚合 | 不由设备直接覆盖平台状态 |
| Snapshot | 无直接字段 | 平台根据状态版本创建，不写入 MQTT |

Action 规则：

- 只有 Platform API 的 Action Service 可以发布 command。
- 前端和 Agent 不直接连接 MQTT。
- Agent 先生成 Plan/Action，Action 必须绑定已存在的 `robotCode`，经过校验后转换为对应机器人的 command topic。
- result event 先保存 MessageRecord，再生成 Observation，再更新 CurrentState。

## 16. 多 Workspace 联调参数

每个 Workspace 的连接信息独立下发：

```text
workspaceId
brokerHost
brokerPort
clientId
username
password/certificate reference
commandTopic
resultTopic
```

其中 `workspaceId` 仅存在于平台联调配置，不进入机器狗 MQTT Payload。

验收要求：

- 两个 Workspace 使用相同 `robotCode` 时消息互不串流。
- 一个 Workspace 停止或 Broker 重启不影响其他 Workspace。
- 机器狗凭证无法连接其他 Workspace Broker。
- command/result 可通过 `taskId/commandId/requestId/traceId` 串联到驾驶舱 Trace。
