| 项目 | 内容 |
|---|---|
| 文档类型 | P3 数据库补充标准 |
| 日期 | 2026-06-23 |

## 1. Schema

P3 新增独立 schema：

```text
simulation
```

该 schema 存放运行模型，不修改 P1 已完成表结构。

## 2. 表结构

P3 首轮新增表：

```text
simulation.simulation_runs
simulation.tasks
simulation.plans
simulation.plan_steps
simulation.actions
simulation.observations
simulation.current_states
simulation.snapshots
simulation.trace_headers
simulation.trace_spans
```

## 3. 表职责

| 表 | 职责 |
|---|---|
| `simulation_runs` | 单次仿真运行容器，绑定场景、地图版本和运行状态 |
| `tasks` | 业务目标、输入、约束、优先级和任务状态 |
| `plans` | Task 的可版本化执行计划 |
| `plan_steps` | Plan 的可执行步骤 |
| `actions` | 一次动作执行尝试，关联 command |
| `observations` | MQTT result、异常注入、控制台事件等事实观察 |
| `current_states` | 按 run 聚合的当前状态投影，带 `state_version` |
| `snapshots` | 指定状态版本的不可变检查点 |
| `trace_headers` | Trace 头信息 |
| `trace_spans` | Trace 过程节点 |

## 4. 关键索引

每张表必须包含：

```text
workspace_id
run_id
```

跨对象追踪依赖：

```text
task_id
plan_id
plan_step_id
action_id
command_id
request_id
trace_id
observation_id
snapshot_id
```

核心唯一约束：

```text
(workspace_id, run_id)
(workspace_id, task_id)
(workspace_id, plan_id)
(workspace_id, plan_step_id)
(workspace_id, action_id)
(workspace_id, observation_id)
(workspace_id, snapshot_id)
(workspace_id, trace_id)
(workspace_id, span_id)
```

## 5. 数据流

```text
simulation_runs
-> tasks
-> plans
-> plan_steps
-> actions
-> message.message_records(command)
-> message.message_records(event)
-> observations
-> current_states
-> snapshots
-> trace_headers / trace_spans
```

## 6. CurrentState 规则

`current_states` 是可重建投影，不是事实源。

事实源优先级：

```text
message.message_records
simulation.observations
simulation.actions
simulation.tasks
simulation.plans
runtime.robot_instances
```

更新规则：

- 每次 Observation 应使 `state_version` 单调递增。
- `task_state_json` 保存当前 Task 摘要，不保存全量历史。
- `active_plan_json` 保存当前 Plan 摘要。
- `robot_states_json` 保存当前机器人投影。
- `resource_states_json` 保存资源锁、阻塞路径、工位状态等。
- `environment_state_json` 保存场景、地图版本、异常摘要等。
- `active_events_json` 保存当前有效异常和事件摘要。

## 7. Snapshot 规则

Snapshot 必须引用明确的：

```text
run_id
state_version
reason
checksum
```

Snapshot 不按固定高频生成，只在以下时机生成：

- Run 创建。
- 人工创建检查点。
- Task 完成。
- 严重异常。
- Run 结束。
